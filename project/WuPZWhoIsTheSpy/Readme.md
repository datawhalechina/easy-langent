# “谁是卧底”游戏智能体

🚀 核心功能
- 用户作为上帝/旁观者，不参与游戏操作，所有游戏参与方均为智能体，全程由智能体自主完成游戏流程，用户仅查看游戏输出结果。
- 状态管理：使用 TypedDict 和 GameState 全程跟踪游戏进度和数据。


游戏过程：
1. 游戏参与方：4个智能体（1个卧底，3个平民）；
2. 词语分配：平民获得相同的“平民词”，卧底获得与平民词相关但不同的“卧底词”（如平民词“奶茶”，卧底词“果汁”）；
3. 游戏流程：发言阶段：4个智能体依次发言，描述自己拿到的词语（不能直接说词语本身）；
4. 投票阶段：每个智能体根据所有发言，投票选出自己怀疑的“卧底”；
5. 胜负判断：得票最多的智能体被淘汰，若淘汰的是卧底，平民获胜；若淘汰的是平民，游戏继续（重复发言-投票流程）；若剩余1个平民和1个卧底，卧底获胜。

🛠️ 技术栈
- 框架: LangGraph, LangChain
- 模型: DeepSeek 
- 状态存储: 内存检查点 (MemorySaver)
- python 版本： 3.12
- 依赖包安装命令：`conda install langgraph langchain langchain_openai langchain_core os sys random json textwrap typing dotenv`
 

📋 工作节点
词语生成模块（generate_words）：随机生成1组平民词和对应的卧底词（确保相关性，如“手机-电话”“米饭-面条”）；
角色分配模块（assign_roles）：将词语分配给3个智能体，随机指定1个为卧底，2个为平民；
发言生成模块（generate_speeches）：智能体根据自己的角色（平民/卧底）和拿到的词语，生成符合规则的发言；
投票模块（vote_undercover）：智能体根据所有发言，分析并投票选出怀疑的卧底；
游戏控制模块(judge_result)：串联所有模块，控制游戏流程（发言→投票→胜负判断），记录游戏状态；
结果展示模块（show_final_result）：输出每一轮的发言、投票结果，以及最终的游戏胜负。

使用方法：
1. API密钥配置: 在当前目录中新建`.env`文件，在文件中配置自己的密钥。默认使用deepseek最新模型，也可修改配置文件使用其他模型。
2. 运行命令：`python WhoIsTheSpy.py`
